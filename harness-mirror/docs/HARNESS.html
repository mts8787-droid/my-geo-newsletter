🐈‍⬛ HIRO

Harness for Interactive Reporting Optimization

Built for beginners. Engineered for repetition.

🐈‍⬛ 히로 (HIRO) — 본 하네스의 마스코트 검은 고양이. 부트스트랩 시나리오 같은 대화형 작업에서 친근한 안내자로 말을 건넵니다.

HIRO optimizes how you build interactive dashboards with Claude Code. Instead of crafting data parsers, chart components, and newsletter templates from scratch every time, HIRO ships a reusable harness — skills, rules, hooks, and scenario-based bootstraps — that turns the manual workflow into a standardized, repeatable pipeline.

HIRO 는 Claude Code 로 인터랙티브 대시보드를 만드는 작업을 최적화합니다. 데이터 파서, 차트 컴포넌트, 뉴스레터 템플릿을 매번 처음부터 짜는 대신, HIRO 는 재사용 가능한 하네스 — 스킬·룰·훅·시나리오 기반 부트스트랩 — 를 제공해 수작업 워크플로우를 표준화된 반복 가능한 파이프라인으로 바꿉니다.

이 페이지가 뭔가요? — Claude 가 새 대시보드 / 뉴스레터 / KPI 리포트를 만들 때 같은 실수를 반복하지 않도록 도와주는 규칙 + 자동 검사 + 작업 매뉴얼 + 보조 일꾼 묶음입니다.

가전 GEO KPI 영역에서 1000+ 회 commit 으로 다듬어진 패턴이에요. 같은 가전 산업 내 영업 · 매출 · 전략 · 마케팅 · R&D 등 다른 직무 도메인에도 데이터 5행만 보여주면 Claude 가 자동 적응해서 셋업해줍니다 (변수명·코드 한 줄 안 봐도 됨).

본 페이지는 사람이 보기 좋게 정리한 미러이고, 실제 Claude 가 읽는 원본은 CLAUDE.md + .claude/ 폴더에 있어요. 원본 수정 시 npm run build 실행 시 본 미러 자동 갱신.

4 가지 종류 (Rule · Hook · Skill · Sub-Agent)

종류한 줄 설명강제력예시
RULE Rule (규칙) Claude 가 코드 작업할 때 따라야 할 약속. 'NEVER ...' 같은 안 되는 것 + 토큰·이름 규약. 권고 (~80%) "카테고리 매핑은 categoryMap.js 한 곳에만"
HOOK Hook (자동 차단) Rule 중 회귀 위험 큰 항목을 시스템이 100% 막아주는 자동 검사. Claude 가 어겨도 시스템이 차단. 100% (시스템 차단) "빌드 산출물 직접 수정" 시도 → 자동 거부
SKILL Skill (작업 매뉴얼) 특정 작업을 step-by-step 으로 진행하는 워크플로우. 자연어로 호출하면 Claude 가 해당 매뉴얼 따라 진행. 권고 (필요 시 로드) "뉴스레터 만들어줘" → newsletter-make 스킬
AGENT Sub-Agent (보조 일꾼) 특정 영역만 보는 보조 에이전트. 메인 Claude 가 위임할 때 활성화. 보통 진단·읽기 전용. 위임 시 활성 data-puller — 데이터 파이프라인 진단 전담
⚠ Hook 만 100% 강제 — Rule / Skill 은 Claude 가 읽고 권고로 따르는 수준. 절대 어기면 안 되는 것은 Hook 으로 만들어야 시스템 차단됨.

폴더 구조 (2벌 정리)

1) 실제 작동 (Claude Code 공식 — Claude 가 자동 로드)

CLAUDE.md (프로젝트 헌법, 항상 로드) AGENTS.md (OpenAI Codex / Antigravity 자동 로드) .claude/ ├── settings.json (Hook 등록 JSON 강제) ├── hooks/ (*.sh + README.md) ├── skills/<name>/SKILL.md (Skill — data, design, newsletter 등) ├── agents/ (Sub-Agent — data-puller.md 등) └── rules/ (data.md, design.md, ai.md 등 — 비공식 컨벤션)

2) 미러 (사람용 진입점 + git 공유 — 본 폴더 harness-mirror/)

harness-mirror/ ├── CLAUDE.md (루트 CLAUDE.md 미러) ├── AGENTS.md (루트 AGENTS.md 미러) ├── .claude/ (실제 .claude/ 1:1 미러 — 그대로 다른 프로젝트에 사용 가능) │ ├── settings.json │ ├── hooks/{*.sh, README.md} │ ├── skills/<name>/SKILL.md │ ├── agents/*.md │ └── rules/*.md └── docs/ (사람용 가이드 모음) ├── HARNESS.md (전체 설명 마크다운) ├── HARNESS.html (본 파일 — 브라우저 더블클릭) ├── CHART_LIBRARY.html (차트 분류 카탈로그 정적 스냅샷) ├── HUMAN_GUIDE.md (사람 사용 설명서) └── hooks/ (Hook 영역별 영향) ├── data.md └── design.md

RULE Rule — 따라야 할 규칙

토큰·invariant·ANTI-PATTERN 정의. Markdown 권고 (~80%).

  • CLAUDE.md — 프로젝트 헌법 (4 개념 정의 + NEVER Rule + 작업 흐름)
  • .claude/rules/data.md — 데이터 작업 Rule·매뉴얼·invariant (5단계 ERROR CATCHING / null vs 0 / 날짜 정규화)
  • .claude/rules/design.md — 디자인 토큰·컴포넌트 카탈로그 (C-01~C-24)·SVG 패턴·이메일 호환 ANTI-PATTERN
  • .claude/rules/newsletter.md — 이메일 호환 (table-layout, 인라인 style), V1/V2/V3 카드, 발송 전 검증
  • .claude/rules/ai.md — Anthropic API / 인사이트 생성 / N3 검증 / 비용 추적
  • .claude/rules/BOOTSTRAP.md · BOOTSTRAP-newsletter.md — Claude 가 따라가는 시나리오

HOOK Hook — 절대 금지 강제

시스템이 직접 차단. JSON 필수 + 인간 가독 md 설명서.

  • docs/hooks/data.md — 데이터 영역 적용 Hook (syntax-check) + 추가 후보
  • docs/hooks/design.md — 디자인 영역 적용 Hook (block-dist) + 추가 후보

원본 정의: .claude/settings.json (JSON 강제), .claude/hooks/*.sh (스크립트), .claude/hooks/README.md (사람용 설명서)

SKILL Skill — 영역별 카테고리

사람이 보기 좋도록 영역별로 묶음. 원본 .claude/skills/ 폴더는 평탄한 구조 (Claude Code 공식 컨벤션). 본 그룹핑은 미러 페이지에만 적용.

스킬 사용법 — Claude 에게 자연어로 "○○ 스킬 써줘" / "○○ 스킬로 △△해줘" 라고 말하거나, 슬래시 명령 /skills ○○ 로 호출. Claude 가 자동 매칭도 함 (예: "뉴스레터 만들어줘" → newsletter-make).

AGENT Sub-Agent

특정 영역 분리 작업 — Claude Code 공식 기능. Claude 가 위임 시 활성화.

  • .claude/agents/data-puller.md — 데이터 파싱 파이프라인의 shape·정합성·누락 조사·보고 (read-only)

다른 프로젝트에 적용

  1. ZIP 다운로드 (어드민 /hiro) 또는 GitHub 리포 (https://github.com/mts8787-droid/HIRO) clone
  2. 대상 프로젝트 루트에 CLAUDE.md, AGENTS.md, .claude/, harness-mirror/ 복사
  3. Hook 실행 권한 부여 — Hook이 실행되려면 필수. 터미널에서 압축을 풀었던 프로젝트 루트 폴더로 이동 (cd /your/project/path) 한 다음 한 줄 실행: 
    chmod +x .claude/hooks/*.sh
    확인: ls -l .claude/hooks/ 가 -rwxr-xr-x 표시하면 OK.
  4. Claude Code 실행 → 4가지 자동 로드 (사용자는 아무것도 안 해도 됨):
    • 규칙 (Rule) — Claude 가 따라야 할 패턴. CLAUDE.md + .claude/rules/
    • 자동 검사 (Hook) — 절대 하면 안 되는 것 시스템 차단 (100% 강제). .claude/settings.json + .claude/hooks/*.sh
    • 작업 순서 (Skill) — "데이터 추가해줘" 같이 말하면 적절한 Skill 자동 호출. .claude/skills/<name>/SKILL.md
    • 보조 일꾼 (Sub-Agent) — 특정 영역 분리 작업. .claude/agents/<name>.md
  5. "이 하네스 적용해줘" 라고 Claude 에게 부탁 — 🐈‍⬛ 히로가 도메인 인터뷰 (2 질문) → "데이터 1~5행 보여주세요" → 자동 분석 + 추론 결과 확인 → 모든 파일 자동 생성. 변수명·코드 한 줄 안 봐도 됨. 같은 가전 산업 내 영업 · 매출 · 전략 · 마케팅 · R&D 등 다른 직무 도메인에도 동일.
💡 harness-mirror/ 는 사람용 설명 사본 — 실제 작동에는 영향 X. 필요 없으면 삭제 가능 (단, GitHub 리포지토리 정리 시 손쉽게 다시 받을 수 있음).

본 저장소 갱신 흐름

  1. 원본 파일 수정 (예: .claude/rules/data.md, .claude/skills/data/SKILL.md)
  2. npm run build 또는 npm run sync:harness → harness-mirror/ 자동 갱신
  3. 미러 폴더 변경도 함께 커밋
💡 자동 동기화 — package.json 의 prebuild 스크립트가 node scripts/sync-harness.mjs 호출. 빌드 시점에 항상 미러 최신.

강제력 격상 패턴

Rule을 권고가 아닌 100% 강제로 만들려면 → 해당 Rule을 검출하는 *.sh Hook 작성 → .claude/settings.json 등록.

예시는 docs/hooks/data.md · docs/hooks/design.md 의 "추가 후보 Hook" 섹션 참고.
