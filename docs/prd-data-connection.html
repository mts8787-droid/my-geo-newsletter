SEC 00

Overview — Data Pipeline Architecture

SEMrush Enterprise API에서 수집한 SEO/SEM 데이터를 Google BigQuery에 저장하는 파이프라인입니다. 아래 6단계를 순서대로 완료하면 데이터 수집이 가능합니다.

SEMrush API Analytics / Projects / Listing
→
Dashboard App Streamlit + Python
→
BigQuery 9 Tables / Auto-schema

End-to-End Flow

GCP Project→
Service Account→
SEMrush Keys→
.env Config→
Test→
Collect!
예상 소요 시간: 처음 설정 시 약 15~20분. GCP 프로젝트가 이미 있다면 10분 이내.
STEP 01

GCP 프로젝트 생성 & BigQuery API 활성화

1-1. GCP 프로젝트 생성

Google Cloud Console 접속

Google 계정으로 console.cloud.google.com에 로그인합니다.

Google Account

새 프로젝트 만들기

상단 프로젝트 선택 드롭다운 → 새 프로젝트 클릭

  • 프로젝트 이름: semrush-dashboard (자유)

이 값이 GCP_PROJECT_ID 환경변수에 들어갑니다.

STEP 02

서비스 계정 생성 & JSON 키 발급

BigQuery에 데이터를 쓰려면 서비스 계정(Service Account)이 필요합니다. 사람이 아닌 앱이 GCP에 인증할 때 사용하는 전용 계정입니다.

2-1. 서비스 계정 생성

IAM 페이지 이동

GCP Console 좌측 → IAM 및 관리자 → 서비스 계정 → + 서비스 계정 만들기

계정 정보 입력

  • 이름: semrush-dashboard
  • ID: 자동 생성 (수정 가능)
  • 설명: SEMrush data pipeline (선택)

만들고 계속 클릭

역할(Role) 부여 — 필수 2개

BigQuery 데이터 편집자 BigQuery 작업 사용자

roles/bigquery.dataEditor — 테이블 읽기/쓰기

roles/bigquery.jobUser — 쿼리 실행 권한

필수
역할 2개 모두 추가해야 합니다. dataEditor만 있으면 쿼리 실행이 안 되고, jobUser만 있으면 테이블에 쓸 수 없습니다.

2-2. JSON 키 다운로드

서비스 계정 클릭

방금 만든 계정의 이메일 클릭

키 탭 → 새 키 만들기

키 탭 → 키 추가 → 새 키 만들기

JSON 선택 → 다운로드

JSON 유형 선택 → 만들기 클릭. 파일이 자동 다운로드됩니다.

보관 필수

STEP 03

SEMrush API 키 & 프로젝트 설정

SEMrush에서 3가지 인증 정보를 수집합니다. 사용하는 기능에 따라 필요한 항목이 다릅니다.

3-1. API Key 발급

SEMrush API Key

모든 API 호출에 사용되는 기본 인증 키입니다.

  • 위치: SEMrush 로그인 → 좌측 하단 Subscription Info → API units 섹션
  • 또는: Settings → API → Show API Key
  • 40자 영숫자 문자열 (예: a1b2c3d4e5f6...)
Analytics API Projects API API Key Auth
필수

3-2. Project ID & Campaign ID

Project ID

Site Audit, Position Tracking 등 프로젝트 기반 기능에 필요합니다.

  • 위치: SEMrush → Projects → 프로젝트 선택
  • URL에서 확인: /projects/12345678/...
  • 숫자로만 구성
Site Audit 사용 시 필수

Campaign ID

Position Tracking (Visibility Index) 기능에 필요합니다.

  • 위치: Projects → Position Tracking 페이지
  • URL에서 확인: /tracking/6647718_272401/...
  • 형식: 프로젝트ID_캠페인숫자
Visibility 사용 시 필수

3-3. Listing Management Token

OAuth Bearer Token

로컬 Citation / Listing Management 기능에 필요한 별도 인증 토큰입니다.

  • SEMrush Listing Management API는 OAuth 2.0 인증 사용
  • SEMrush 지원팀에 API 접근 권한 요청 필요
  • 발급받은 Bearer Token을 SEMRUSH_LISTING_TOKEN에 설정
Listing Management API OAuth Bearer
Citations 사용 시 필수

API 인증 요약

API 인증 방식 환경변수 용도 필수 여부
Analytics API API Key (Query Param) SEMRUSH_API_KEY 도메인 분석, 키워드, 백링크, 경쟁사 필수
Projects API API Key (Query Param) SEMRUSH_API_KEY + PROJECT_ID Site Audit, Position Tracking 선택
Listing API OAuth Bearer Token SEMRUSH_LISTING_TOKEN 로컬 리스팅, NAP 데이터, Citations 선택
STEP 04

환경변수 설정

수집한 모든 키와 설정값을 .env 파일에 입력합니다. 두 가지 배포 환경에 따라 방법이 다릅니다.

4-1. 로컬 환경 — .env 파일

.env # ── SEMrush API (필수) ──
SEMRUSH_API_KEY=your_api_key_here

# ── SEMrush Enterprise (선택) ──
SEMRUSH_PROJECT_ID=12345678
SEMRUSH_CAMPAIGN_ID=6647718_272401
SEMRUSH_LISTING_TOKEN=your_oauth_token

# ── Google Cloud / BigQuery (필수) ──
GCP_PROJECT_ID=semrush-dashboard-438012
BQ_DATASET=semrush_data
GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account-key.json

# ── 기본 설정 ──
SEMRUSH_DATABASE=us # us, kr, jp, uk 등
TARGET_DOMAIN=example.com

4-2. Render 배포 환경

Render에서는 파일 경로 대신 JSON 문자열을 환경변수로 전달합니다.

JSON 키를 문자열로 변환

다운로드한 JSON 파일 내용을 한 줄로 만듭니다.

# 터미널에서 실행
cat service-account-key.json | tr -d '\n'

Render 환경변수에 입력

Render Dashboard → Service → Environment 탭

  • GOOGLE_APPLICATION_CREDENTIALS_JSON = (JSON 문자열 전체)
  • 나머지 변수도 동일하게 추가
로컬 vs Render 인증 차이:
• 로컬: GOOGLE_APPLICATION_CREDENTIALS — JSON 파일 경로 지정
• Render: GOOGLE_APPLICATION_CREDENTIALS_JSON — JSON 내용을 문자열로 전달 (앱이 임시파일로 자동 변환)

전체 환경변수 레퍼런스

환경변수 설명 예시 값 기본값 필수
SEMRUSH_API_KEY SEMrush API 인증 키 a1b2c3d4e5... — 필수
GCP_PROJECT_ID GCP 프로젝트 ID my-project-438012 — 필수
GOOGLE_APPLICATION_CREDENTIALS 서비스 계정 키 파일 경로 ./keys/sa.json — 로컬
GOOGLE_APPLICATION_CREDENTIALS_JSON 서비스 계정 키 JSON 문자열 {"type":"service_account",...} — Render
BQ_DATASET BigQuery 데이터셋 이름 semrush_data semrush_data 기본값 있음
SEMRUSH_DATABASE SEMrush 국가 DB 코드 kr us 기본값 있음
TARGET_DOMAIN 분석 대상 도메인 lg.com example.com 기본값 있음
SEMRUSH_PROJECT_ID SEMrush 프로젝트 번호 12345678 — 선택
SEMRUSH_CAMPAIGN_ID Position Tracking 캠페인 6647718_272401 — 선택
SEMRUSH_LISTING_TOKEN Listing API OAuth Token eyJhbGci... — 선택
STEP 05

연결 테스트

환경변수 설정이 끝나면 앱을 실행하여 연결을 확인합니다.

앱 실행

streamlit run app.py

Settings 페이지 이동

사이드바 → Settings 페이지 → 전체 연결 상태 진단 버튼 클릭

결과 확인

4개 항목 모두 ✓ 표시되면 성공:

  • GCP 프로젝트 ID
  • 서비스 계정 키
  • BigQuery 연결
  • SEMrush API Key
웹 UI에서도 설정 가능: Settings 페이지에서 JSON 파일 업로드 또는 붙여넣기로 세션 내 즉시 연결 테스트가 가능합니다. .env 파일 없이도 먼저 테스트해볼 수 있습니다.
STEP 06

첫 데이터 수집

연결이 확인되면 두 가지 방법으로 데이터를 수집할 수 있습니다.

방법 A: 웹 대시보드

Streamlit 앱에서 각 페이지별로 데이터 조회 → BigQuery 저장

Page 1
VisibilityVisibility Index + 키워드 순위 → BigQuery
Page 2
Citations로컬 리스팅 / NAP 데이터 → BigQuery
Page 3
Site Audit건강도 스냅샷 → BigQuery
Page 4
Analytics도메인/키워드/백링크/경쟁사 → BigQuery

방법 B: CLI 파이프라인

터미널에서 명령어로 일괄 수집

# 전체 수집
python main.py --domain lg.com --all

# 개별 수집
python main.py --domain lg.com --visibility
python main.py --domain lg.com --analytics
python main.py --domain lg.com --audit
python main.py --domain lg.com --citations

# 한국 DB 사용
python main.py --domain lg.com --all --database kr

BigQuery 테이블 — 자동 생성

데이터 수집 시 아래 9개 테이블이 자동으로 생성됩니다. 스키마도 자동 감지(autodetect)됩니다.

테이블명 설명 소스 API 수집 페이지
visibility_index Visibility Index 추이 Projects API Page 1
position_tracking 키워드 순위 추적 Projects API Page 1
site_audit 사이트 건강도 스냅샷 Projects API Page 3
citations 로컬 Citation/리스팅 Listing API Page 2
domain_overview 도메인 개요 (트래픽, 랭크) Analytics API Page 4
domain_organic_keywords 오가닉 키워드 (최대 10K행) Analytics API Page 4
domain_adwords_keywords 유료 키워드 (최대 10K행) Analytics API Page 4
backlinks_overview 백링크 통계 Analytics API Page 4
organic_competitors 오가닉 경쟁사 Analytics API Page 4
메타 컬럼 자동 추가: 모든 테이블에 _loaded_at (수집 시간)과 _source (semrush_enterprise) 컬럼이 자동으로 추가됩니다. BigQuery 관리 페이지(Page 5)에서 테이블 현황 확인 및 SQL 쿼리가 가능합니다.

SEC 08

Setup Checklist

아래 체크리스트를 순서대로 완료하세요. 완료된 항목은 클릭하여 체크할 수 있습니다.

  • GCP 프로젝트 생성 완료 & 프로젝트 ID 메모
  • BigQuery API 활성화
  • 서비스 계정 생성 (dataEditor + jobUser 역할)
  • JSON 키 파일 다운로드 & 안전한 위치에 보관
  • SEMrush API Key 확인
  • SEMrush Project ID / Campaign ID 확인 (사용하는 경우)
  • .env 파일 작성 완료 (또는 Render 환경변수 설정)
  • 앱 실행 & Settings 페이지에서 연결 진단 통과
  • 첫 데이터 수집 성공 (Web UI 또는 CLI)
  • BigQuery 관리 페이지에서 테이블 & 데이터 확인
