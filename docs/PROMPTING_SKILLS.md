# Prompting Skills — 본 프로젝트 개발 시 사용한 프롬프팅 패턴

GEO Newsletter / Dashboard / Report 시스템을 Claude(개발 보조)와 함께 구축하면서 실제로 효과적이었던 프롬프팅 기법을 **대시보드 기능별**로 정리.

각 사례는 "사용자가 어떤 식으로 요청했고", "왜 그것이 효과적이었는지"를 함께 기록.

---

## 1. Visibility — 제품 카드 · 국가 필터 · MoM/WoW

### 기능 요약

- 제품별 카드 (LG 점수, 경쟁비, 신호등, 미니 트렌드)
- 국가별 vbar 차트 (LG vs 경쟁사 vs C-Brand)
- 국가 필터 / BU 필터 / 월 드롭다운 → 카드 점수·MoM 동적 재계산

### 효과적이었던 프롬프트 패턴

**(1) 기대값을 구체적 수치로 명시**
```
USA만 선택 시 USA TV 점수는 88.1% 가 맞나요?
- 88.1% (Apr)
- 88.9% (Mar)
- 87.1% (Feb)
- 다른 값 (정확히 얼마?)
```
- 객관식 + "다른 값(정확히 얼마)" 옵션 → 의도 명확화 + 디버깅 분기 단순화

**(2) 데이터 원형 직접 공유**
```
Div    Date    Country    Category    LG    SAMSUNG    ...
MS    26년 2월    TTL    TV    86.0%    91.6%    ...
MS    26년 2월    US    TV    87.1%    87.2%    ...
```
- 시트 컬럼 헤더 + 데이터 행 2~3개 → 파싱 로직 검증 가능
- 한국식 `26년 2월` 포맷이 `parseMonthFromDate` 의 `\d{4}` 정규식과 안 맞는 것을 즉시 발견

**(3) "주간은 잘 되는데 월간이 안 됨" — 참조 패턴 명시**
```
주간에서는 잘되는데 주간 계산코드를 참고해서 월간을 잘짜봐
```
- 동일한 패턴을 다른 영역에 복제하라는 명시 → 일관성 보장

**(4) 콘솔 명령 결과 회신**
```
_filteredMomD('tv', ['US']) → -0.8
_filteredMomD('tv', ['US','CA']) → -4.9
prod-mom 표시값 → 'MoM ▼ 1.5%p'  ← 불일치!
```
- 함수 반환값 vs DOM 표시값 비교 → "함수 정상, DOM 업데이트 누락" 으로 좁힘

## 2. Citation — 트렌드 / Bump Chart / 국가 필터

### 기능 요약

- 도메인별 / 카테고리별 Citation Bump Chart (월별 랭킹 변동)
- 국가 필터 적용 시 선택 국가들 값 합산
- 클라이언트 인라인 SVG 생성 (서버 사전 계산 + 클라이언트 재집계)

### 효과적이었던 프롬프트 패턴

**(1) 단순화 요청**
```
간단하게 전월과의 차이만 보면 되
```
- 복잡한 가중치/추세 분석 대신 단순 diff 로 좁힘
- 첫 구현은 복잡할수록 망가질 가능성 ↑

**(2) 상위 데이터 흐름 짚어주기**
```
'TTL|*' 키는 서버에서 미리 계산된 전체 합계.
국가 필터 적용 시 '${cnty}|*' 키들을 다시 합산해야 해.
```
- 데이터 구조를 사용자가 직접 짚어줘서 클라이언트 재집계 로직 빠르게 설계

## 3. 베이스라인 리셋 마커 (Audio · RAC · Aircare)

### 기능 요약

- 특정 제품의 특정 시점부터 baseline 재설정 (Audio W13, RAC/Aircare W16)
- 차트에서 이전 기간 회색 페이드 + 새 시점에 dashed vertical + 라벨

### 효과적이었던 프롬프트 패턴

**(1) 점진적 조정 (10% / 5px / 25px 단위)**
```
RAC와 에이케어 제품카드의 Baseline 글씨를 25px 아래로 낮춰줘
... (테스트 후)
Aircare 주간 아래로 10px 더 내려줘
... (테스트 후)
Audio 월간 -40px 더 올려줘
```
- 작은 단위 반복 → 정확한 위치 찾기
- 매번 재게시·시각 확인 → 인지부하 ↓ vs 한 번에 큰 변경

**(2) 상대적 위치 설명**
```
주차표시 라벨과 실제 데이터 값 사이의 높이정도가 좋아
```
- 절대 좌표 대신 "두 요소 사이" 같은 상대 표현 → 다양한 데이터에 robust

**(3) 차별화 명시**
```
Audio는 W12-W13 회색 연결, RAC/Aircare는 W15-W16 끊김
```
- 처음에는 동일 패턴 → 후에 차이 발견 → 명시적으로 분기 요청

## 4. SVG 차트 (svgLine · svgMultiLine · 클라이언트 _miniSvg)

### 기능 요약

- 서버 사이드 SVG 문자열 생성 (`dashboardSvg.js`)
- 클라이언트에서 필터 변경 시 같은 알고리즘으로 재렌더 (`dashboardClient.js` 내 `_miniSvg`, `_miniSvgNullAware`)

### 효과적이었던 프롬프트 패턴

**(1) 화면 캡처 + 텍스트 보완**
```
[스크린샷] 막대 그래프 밑에 브랜드명이 가끔 짤려
(밑줄친 브랜드 - 하이센스. 블루에어, 필립스 등) 짤리지않게 조정해줘
```
- 시각 컨텍스트 + 어떤 요소가 잘리는지 텍스트 명시 → 즉시 CSS `overflow:hidden`/`width:26px` 원인 짚음

**(2) "어디서 어떻게 틀린지 표 형식으로"**
```
| 단계 | TV 점수 | TV MoM |
|---|---|---|
| 1. 전체 국가 | 88.7% | ▲1.2%p |
| 2. USA 만 | ? | ? |
```
- 빈 칸 채우기 형식 → 사용자도 검증하기 쉬움

## 5. 뉴스레터 컨테이너 너비 / 정렬

### 기능 요약

- 이메일 HTML 호환 테이블 레이아웃
- 920px 컨테이너 + 좌우 그레이 여백
- align="center" + 외곽 padding

### 효과적이었던 프롬프트 패턴

**(1) 우선순위 명확화**
```
align은 안건드리는게 좋겠어
```
- 이전 시도가 어긋났을 때 "건드리지 마" 라는 제약 → 다른 방향 탐색 강제

**(2) 점진 + 회귀**
```
920 → 950 → 980 (확장)
짤림 보고 → 920 (회귀)
940 (절충)
```
- 너무 많이 확장 → 짤림 발견 → 회귀 후 절충점 찾기

**(3) 작업 전/후 비교**
```
오히려 작업전보다 더 오른쪽이 짤려
```
- 단순 회귀 신호 → 직전 변경 revert

## 6. 데이터 파싱 (Google Sheets / 한국식 날짜)

### 기능 요약

- Google Sheets API 동기화
- 다양한 날짜 포맷: `26년 2월`, `Apr 2026`, `2026-04`, `W13`
- `parseMonthFromDate` (서버) + `_dateMi` (클라이언트)

### 효과적이었던 프롬프트 패턴

**(1) 데이터 한 줄만 공유**
```
시트의 date 컬럼은 '26년 2월' 형식이야
```
- 한 줄 공유 → `(\d{2})년` 정규식 누락 즉시 발견

**(2) 시각적 결과 회신**
```
26년 2월 → score: 87.1
26년 3월 → score: 88.9
26년 4월 → score: 88.1
```
- 정렬 후 결과 직접 회신 → 서버 sort 로직 정확성 검증

## 7. AI 인사이트 (Anthropic SDK Streaming)

### 기능 요약

- 제품·국가·Citation 등 섹션별 AI 텍스트 생성
- Anthropic SDK `messages.stream().finalMessage()` 사용
- 10분 이상 걸리는 응답 지원

### 효과적이었던 프롬프트 패턴

**(1) 에러 메시지 그대로 공유**
```
403 streaming required
```
- SDK 에러 원본 → 즉시 streaming 전환 필요성 식별

**(2) 출력 형식 제약**
```
전월비를 비중(%) 으로만 작성해. 절대 건수 사용 금지.
```
- AI 출력 형식 강제 → 일관성 보장

## 8. 미출시 국가 / 베이스라인 리셋 안내 문구

### 기능 요약

- `unlaunchedMap` 기반 자동 각주 생성
- 베이스라인 리셋 제품(Audio/RAC/Aircare) 안내 문구

### 효과적이었던 프롬프트 패턴

**(1) 정확한 문구 직접 전달**
```
*Baseline 재조정 (4월)
-Audio : 오디오 신제품 Sound Suite의 브랜드 전략 및 ...
-RAC/Aircare : 사업 중요도에 따라서 ...
```
- 사용자가 최종 문구 작성 → 번역/요약 오류 방지

**(2) 형식 디테일 별도 요청**
```
컴마 뒤에 띄어쓰기 삭제함
Sound Suit 이라고 되어있는데 오타야 Sound Suite 으로 다 바꿔줘
```
- 본문 작성 후 → 띄어쓰기·오타 별도 정정 요청

## 9. 게시(Publish) 워크플로우

### 기능 요약

- 동기화(Google Sheets → 서버 데이터)
- 게시(서버 데이터 → 정적 HTML)
- IP 화이트리스트 기반 접근 제어
- 뉴스레터 월별 정적 페이지 자동 생성

### 효과적이었던 프롬프트 패턴

**(1) 환경 명시**
```
어드민에서만 해야해
```
- 게시 후 페이지 vs 어드민 iframe 구분 → 적절한 디버그 컨텍스트 선택

**(2) 재게시 절차 안내**
- "동기화 후 게시" 시퀀스를 사용자가 인지 → 캐시 문제 빠른 해결

## 10. 공통 / 횡단 패턴

### 효과적이었던 일반 패턴

| 패턴 | 예시 |
|---|---|
| **구체적 수치/단위** | "25px", "10%", "30px 여백" |
| **상대적 위치** | "X축 라벨과 데이터 사이 높이" |
| **참조 패턴 지정** | "주간 코드 참고해서" |
| **데이터 원형 공유** | 시트 행 직접 복사 |
| **스크린샷 + 보완** | 짤린 부분 + "밑줄친 브랜드" |
| **제약/금지 명시** | "align은 건드리지 마" |
| **점진 + 회귀** | 920→980→920→940 |
| **콘솔 명령 회신** | DevTools 출력 직접 공유 |
| **빈칸 채우기 표** | 사용자가 기대값 명시 |
| **에러 원본 공유** | "403 streaming required" |
| **단순화 요청** | "단순히 차이만" |

### 비효과적이었던 패턴 (피해야 할 것)

- "다 안 맞아" — 어떤 것이 어떻게 안 맞는지 없음 → 추측에 의존
- "잘 됐어?" → 검증 절차 필요 (사용자가 직접 확인 의지 필요)
- 큰 단위 변경 후 한 번에 검증 → 회귀 원인 찾기 어려움

## 메타 인사이트

본 프로젝트에서 사용자가 가장 자주 사용한 효과적인 프롬프트는:

1. **시각 컨텍스트(스크린샷) + 위치 텍스트** → AI 의 시야 확장
2. **점진적 단위 조정 (5/10/25/30px 단위)** → 정확도 + 검증 용이성
3. **콘솔 명령 결과 직접 회신** → 함수 vs DOM 분리 진단
4. **데이터 한 줄/한 행 직접 공유** → 파서 정합성 즉시 검증
5. **참조 패턴 지정** ("주간 보고 월간 짜")  → 일관성 + 메모리 절약
6. **제약 명시** ("align 건드리지 마") → 탐색 공간 축소

이 6가지 패턴이 본 프로젝트 가속화의 핵심.
