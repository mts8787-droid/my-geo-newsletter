Harness

본 프로젝트의 Claude Code 하네스 (Rule·Hook·Skill·Sub-Agent) 전체 설명

이 페이지가 뭔가요? — Claude 가 본 저장소에서 일할 때 자동으로 따르는 규칙·자동 검사·작업 매뉴얼·보조 일꾼의 묶음입니다. 이 페이지는 사람이 보기 좋게 정리한 미러이고, 실제 Claude 가 읽는 원본은 CLAUDE.md + .claude/ 폴더에 있어요.

원본을 수정하면 npm run build (또는 npm run sync:harness) 실행 시 본 미러가 자동으로 갱신됩니다 — 미러는 직접 수정 X.

4 가지 종류 (Rule · Hook · Skill · Sub-Agent)

종류한 줄 설명강제력예시
RULE Rule (규칙) Claude 가 코드 작업할 때 따라야 할 약속. 'NEVER ...' 같은 안 되는 것 + 토큰·이름 규약. 권고 (~80%) "카테고리 매핑은 categoryMap.js 한 곳에만"
HOOK Hook (자동 차단) Rule 중 회귀 위험 큰 항목을 시스템이 100% 막아주는 자동 검사. Claude 가 어겨도 시스템이 차단. 100% (시스템 차단) "빌드 산출물 직접 수정" 시도 → 자동 거부
SKILL Skill (작업 매뉴얼) 특정 작업을 step-by-step 으로 진행하는 워크플로우. 자연어로 호출하면 Claude 가 해당 매뉴얼 따라 진행. 권고 (필요 시 로드) "뉴스레터 만들어줘" → newsletter-make 스킬
AGENT Sub-Agent (보조 일꾼) 특정 영역만 보는 보조 에이전트. 메인 Claude 가 위임할 때 활성화. 보통 진단·읽기 전용. 위임 시 활성 data-puller — 데이터 파이프라인 진단 전담
⚠ Hook 만 100% 강제 — Rule / Skill 은 Claude 가 읽고 권고로 따르는 수준. 절대 어기면 안 되는 것은 Hook 으로 만들어야 시스템 차단됨.

폴더 구조

원본 (Claude Code 공식 — 실제 작동)

CLAUDE.md .claude/ ├── settings.json (Hook 등록 JSON 강제) ├── hooks/ (syntax-check.sh, block-dist.sh, README.md) ├── skills//SKILL.md (data, design, prompting) ├── agents/ (data-puller.md) └── rules/ (data.md, design.md) ← 비공식 컨벤션

미러 (사람용 진입점 — 자동 생성)

docs/agents/ ├── HARNESS.md ← 전체 설명 (마크다운) ├── HARNESS.html ← 본 파일 (브라우저로 더블클릭) ├── CLAUDE.md ← 프로젝트 헌법 미러 ├── rules/ ← Rule 매뉴얼 미러 │ ├── data.md │ └── design.md ├── skills/ ← Skill 워크플로우 미러 │ ├── data.md │ └── design.md └── hooks/ ← Hook 영역별 관점 ├── data.md └── design.md

RULE Rule — 따라야 할 규칙

토큰·invariant·ANTI-PATTERN 정의. Markdown 권고 (~80%).

  • CLAUDE.md — 프로젝트 헌법 (4 개념 정의 + NEVER Rule + 작업 흐름)
  • rules/data.md — 데이터 작업 Rule·매뉴얼·invariant (5단계 ERROR CATCHING / null vs 0 / 날짜 정규화)
  • rules/design.md — 디자인 토큰·컴포넌트 카탈로그 (C-01~C-23)·SVG 패턴·이메일 호환 ANTI-PATTERN

원본: CLAUDE.md, .claude/rules/data.md, .claude/rules/design.md

HOOK Hook — 절대 금지 강제

시스템이 직접 차단. JSON 필수 + 인간 가독 md 설명서.

  • hooks/data.md — 데이터 영역 적용 Hook (syntax-check 위주) + 추가 후보 (warn-inline-mapping, warn-silent-fallback 등)
  • hooks/design.md — 디자인 영역 적용 Hook (block-dist 위주) + 추가 후보 (block-hardcoded-color, block-flex-in-email 등)

원본 정의: .claude/settings.json (JSON 강제), .claude/hooks/*.sh (스크립트)

SKILL Skill — 영역별 카테고리

사람이 보기 좋도록 영역별로 묶음. 원본 .claude/skills/ 폴더는 평탄한 구조 (Claude Code 공식 컨벤션). 본 그룹핑은 미러 페이지에만 적용.

스킬 사용법 — Claude 에게 자연어로 "○○ 스킬 써줘" / "○○ 스킬로 △△해줘" 라고 말하거나, 슬래시 명령 /skills ○○ 로 호출. Claude 가 자동 매칭도 함 (예: "뉴스레터 만들어줘" → newsletter-make).

AGENT Sub-Agent

특정 영역 분리 작업 — Claude Code 공식 기능. Claude 가 위임 시 활성화.

  • data-puller.md — 데이터 파싱 파이프라인의 shape·정합성·누락 조사·보고 (read-only)

다른 프로젝트에 적용

  1. ZIP 다운로드 (어드민 /admin/harness) 또는 .claude/agents/ + CLAUDE.md + docs/ + .claude/hooks/ + .claude/skills/ + .claude/settings.json 복사
  2. chmod +x .claude/hooks/*.sh
  3. Claude Code 실행 → 자동 로드

본 저장소 갱신 흐름

  1. 원본 파일 수정 (예: .claude/rules/data.md, .claude/skills/data/SKILL.md)
  2. npm run build 또는 npm run sync:harness → .claude/agents/ 미러 자동 갱신
  3. 미러 폴더 변경도 함께 커밋
💡 자동 동기화 — package.json 의 prebuild 스크립트가 node scripts/sync-harness.mjs 호출. 빌드 시점에 항상 미러 최신.

강제력 격상 패턴

Rule을 권고가 아닌 100% 강제로 만들려면 → 해당 Rule을 검출하는 *.sh Hook 작성 → .claude/settings.json 등록.

예시는 hooks/data.md · hooks/design.md 의 "추가 후보 Hook" 섹션 참고.
