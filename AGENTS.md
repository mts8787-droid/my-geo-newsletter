# Agent Harness Engineering & 지식 통합 가이드

## 1. 개요 및 목적
본 문서는 서로 다른 AI 에이전트(Claude, Antigravity 등)가 프로젝트에 투입될 때, **일관된 행동 규범과 코드 품질을 유지하기 위한 '하네스 엔지니어링(Harness Engineering)'의 구현 목적과 방안**을 정리한 것입니다.

## 2. 핵심 구성 요소의 이해

### A. 클로드.md (Claude.md)
* **정의:** Claude 기반 에이전트를 위한 맞춤형 시스템 프롬프트 및 프로젝트 지침 문서입니다.
* **역할:** 프로젝트의 도메인 지식, 코딩 컨벤션, 특정 프레임워크나 디자인 룰(예: PPT Design System) 등 에이전트가 반드시 지켜야 할 구체적인 제약 사항을 정의합니다.

### B. Antigravity Knowledge Item (KI)
* **정의:** Antigravity 에이전트의 영구적 지식 저장 시스템(`appDataDir/knowledge`)입니다.
* **역할:** 과거의 대화 기록, 프로젝트의 아키텍처 패턴, 템플릿 룰 등을 메타데이터(`metadata.json`)와 함께 보존합니다. 에이전트가 새로운 작업을 시작하기 **최우선으로(Mandatory First Step)** 로드하여 중복 작업을 방지하고 일관성을 유지하도록 강제합니다.

### C. OpenAI Codex `AGENTS.md` (KI 등가물)
* **정의:** OpenAI Codex (CLI / Workspace / Cloud) 가 워크스페이스 루트의 **`AGENTS.md`** 파일을 자동 로드하는 표준 컨벤션입니다. Antigravity 의 KI 와 동일한 역할 — 별도의 영구 지식 시스템이 아니라, 루트의 `AGENTS.md` 자체가 영구 컨텍스트 주입 메커니즘으로 동작합니다.
* **역할:** 새 세션이 시작될 때 Codex 가 `AGENTS.md` 를 시스템 프롬프트의 일부로 자동 주입합니다. 사용자가 매번 지침을 반복하지 않아도 에이전트가 본 저장소의 행동 규범을 인지합니다.
* **호환:** 본 통합 가이드를 루트에 `AGENTS.md` (대문자 복수형) 로 두면 **OpenAI Codex 와 Antigravity 양쪽에서 자동 인식·로드** 됩니다. Claude Code 는 별도로 `CLAUDE.md` 를 자동 로드하므로, 본 저장소는 두 파일을 병행 유지합니다 (CLAUDE.md = 프로젝트 구체 지침 / AGENTS.md = 에이전트 공통 통합 가이드).

## 3. Andrej Karpathy 관점의 개발 원칙 (Karpathy Guidelines)
AI(LLM) 코딩 과정에서 흔히 발생하는 실수(오버엔지니어링, 불필요한 리팩토링 등)를 방지하기 위해 Andrej Karpathy가 강조한 보수적이고 신중한 개발 원칙입니다.

1. **Think Before Coding (코딩 전 생각하기)**
   * 함부로 가정하지 않고, 모호한 부분이나 여러 해석이 존재할 경우 반드시 사용자에게 질문합니다.
2. **Simplicity First (단순함 우선)**
   * 문제를 해결하는 '최소한의 코드'만 작성합니다. 요구하지 않은 확장성이나 방어 로직(추측성 기능)을 추가하지 않습니다.
3. **Surgical Changes (외과 수술적 변경)**
   * 수술처럼 정확히 필요한 부분만 건드립니다. 기존의 코드 스타일을 완벽히 유지하며, 문제가 없는 주변 코드의 포맷팅이나 리팩토링을 임의로 진행하지 않습니다. 내 수정으로 인해 발생한 고아(Orphan) 코드만 정리합니다.
4. **Goal-Driven Execution (목표 주도 실행)**
   * 명확한 성공 기준(Success Criteria)을 세우고, 코드를 작성하기 전에 목표를 검증할 방법(테스트 등)을 먼저 정의한 후 루프를 돕니다.

## 4. 통합 전략: `agent.md`를 통한 동일한 하네스 엔지니어링 구축

클로드.md의 '프로젝트 구체적 지침'과 Karpathy의 '보수적 개발 4대 원칙'을 융합하여 하나의 통합 지침서인 **`agent.md`**를 생성합니다. 그리고 이를 Antigravity의 **Knowledge Item(KI)** 시스템에 주입합니다.

### 💡 하네스 엔지니어링(Harness Engineering)의 기대 효과
* **에이전트의 파편화 방지:** Claude를 사용하든, Antigravity를 사용하든 동일한 제어 장치(Harness) 안에서 움직이게 되어 **어떤 에이전트를 호출하든 완벽하게 동일한 수준의 코드 퀄리티와 보수적인 접근 방식**을 보장받을 수 있습니다.
* **Surgical & Safe Execution:** 에이전트가 임의로 전체 시스템을 뒤엎거나 불필요한 코드를 양산하는 것을 원천 차단합니다.
* **영구적 컨텍스트 주입:** KI 시스템에 `agent.md`가 등록됨으로써, 사용자가 매 대화마다 지침을 반복해서 프롬프팅할 필요 없이 에이전트 스스로 본인의 행동을 제약하고 모니터링하게 됩니다.
